# CloakBrowser-Manager 运行指南

## 项目结构

- backend/    — FastAPI (Python) 后端，端口 8080
- frontend/   — React + Vite + Tailwind 前端，端口 5173
- CloakBrowser/ — CloakBrowser 核心库（子项目）

## 方式一：本地开发模式

需要开两个终端，分别运行后端和前端。

### 终端 1 — 启动后端

```bash
cd /Users/jamesyee/Projects/CloakBrowser-Manager
source backend/.venv/bin/activate
CLOAKBROWSER_DATA_DIR=/Users/jamesyee/Projects/CloakBrowser-Manager/.data uvicorn backend.main:app --reload --port 8080
```

### 终端 2 — 启动前端

```bash
cd /Users/jamesyee/Projects/CloakBrowser-Manager/frontend
npm run dev
```

### 访问地址

- 前端：http://localhost:5173
- 后端 API：http://localhost:8080
- API 代理：http://localhost:5173/api/*（Vite 自动代理到后端）

### 限制

本地开发模式下，浏览器启动/VNC 查看功能不可用（需要 KasmVNC、Xvnc 等 Linux 专用组件）。

---

## 方式二：Docker 部署（完整功能，推荐）

一条命令搞定，包含 VNC 浏览器查看等全部功能：

```bash
cd /Users/jamesyee/Projects/CloakBrowser-Manager
docker compose up --build
```

访问地址：http://localhost:8080

如需后台运行：

```bash
docker compose up --build -d
```

停止服务：

```bash
docker compose down
```

---

## 两种方式对比

| | 本地开发 | Docker |
|---|---|---|
| 启动命令 | 两个终端分别启动 | 一条命令 |
| 前端地址 | http://localhost:5173 | http://localhost:8080 |
| 热更新 | 支持，代码修改即时生效 | 不支持，需重新构建 |
| 浏览器启动/VNC | 不可用 | 完整功能 |
| 适用场景 | 开发调试前端/后端 API | 实际使用、完整测试 |

## 环境变量说明

| 变量 | 说明 | 默认值 |
|---|---|---|
| CLOAKBROWSER_DATA_DIR | 数据存储目录（SQLite 数据库、浏览器配置文件） | /data |
| AUTH_TOKEN | 可选认证令牌，设置后所有 /api/* 路由需要认证 | 空（无认证） |

---

## 可选：设置认证令牌

Docker 部署时：

```bash
docker compose up --build -e AUTH_TOKEN=your-secret-token
```

或在 docker-compose.yml 中添加：

```yaml
environment:
  - AUTH_TOKEN=your-secret-token
```
